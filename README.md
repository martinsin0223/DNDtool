# 战争迷雾地图工具

一个给 DND / TRPG 跑团 DM 使用的单文件网页地图工具。打开网页就能用,玩家看着同一块屏幕,DM 在上面一点点擦开战争迷雾、移动 Token、亮出线索。

**在线使用：**  
[https://martinsin0223.github.io/DNDtool/](https://martinsin0223.github.io/DNDtool/)

当前版本：**v3.20.1**

---

## 一分钟先试一下

如果你不想先读说明,可以直接照这 5 步:

1. 打开上面的在线地址。
2. 点顶栏「编辑」→「加载地图」,选一张 PNG / JPG / WebP。
3. 选择「全部覆盖」让整张地图先盖上迷雾。
4. 点「回到跑团」。
5. 用顶栏的「揭」笔刷在地图上一点点擦,玩家就看到了。

试完觉得有意思,再继续往下读怎么把它用进真实跑团。

---

## 这个工具解决什么问题

线下跑团 DM 有一张地图,玩家围着电视、投影或共享笔记本一起看。DM 需要稳定、快速、可控地揭开玩家应该看到的区域,以及在地图上放 Token、线索、标记。

工具就是为这一件事设计的。

### 它坚持的几个原则

- **地图优先**：地图永远是屏幕主角,UI 只服务现场操作,不抢戏。
- **DM 手动控制**：不做自动视野、不做触发器、不做规则判定。该亮哪一块由你决定。
- **共享屏幕可信**：默认是"玩家也能看的现场态",不暴露隐藏对象列表,不暴露 DM 备忘。
- **本地优先**：不需要登录,不需要服务器,网页打开就能用。
- **静态部署**：公开版只有 `README.md` 和 `index.html`,可以放在 GitHub Pages 这类静态托管上,也可以下载到本地双击打开。
- **可搬家**：本机保存、导出场景包、私有云存档三条路径并存,换电脑、出门跑团、回家继续都能接得上。

### 它不做什么

- 不做玩家账号,不做 DM / 玩家权限分离。
- 不做多人实时同步(玩家不能远程看,远程跑团请用 VTT)。
- 不做自动视野(谁看见什么由 DM 决定)。
- 不做规则自动化(没有掷骰、没有 HP、没有状态条)。
- 不导出 PNG 成图(需要图就用系统截图)。
- 不依赖后端、React、npm、CDN 或任何外部资源。

这些不是暂时没做,而是产品边界。如果你需要其中任何一项,你需要的应该是完整 VTT,不是这个工具。

### 它适合谁

- 线下跑团的 DM
- 用电视、投影、平板或共享电脑展示地图的桌面团
- 想要"比截图灵活,但比完整 VTT 轻很多"的地图展示工具
- 需要提前准备多场景、事件点、Token、文字标注、图片标注和遮罩的战役主持人
- 跑团时需要根据玩家拿到的情报,一键揭开预设地图区域的 DM

## 跑团准备：第一次完整流程

下面把准备阶段串成一条线。新 DM 跟着做一遍,大致就懂了。每一步后面有详细章节可以回来翻。

1. **加载地图**(见「准备:加载第一张地图」)
2. **建场景**(如果同一战役有多张地图,见「准备:多场景管理」)
3. **配网格**(如果你想按格揭示或对齐原图网格,见「准备:网格设置 / 网格校准」)
4. **放 Token / 事件 / 标注**(玩家、NPC、线索、机关、地名,见各自章节)
5. **创建词条**(给"亡灵线索""城堡二层"这种语义打标签,见「准备:词条」)
6. **预设揭示区域**(v3.20 新功能。玩家拿到某张旧地图后一键亮起对应区域,见「准备:揭示区域」)
7. **测试一遍现场流程**(切到跑团态,试着用揭、盖、词条、切场景,看屏幕是否清爽)
8. **导出备份 / 上传云存档**(出门前的安全网,见「跨设备:导出 / 导入」和「跨设备:云存档」)

### 一个最常见的工作分配

- **家里准备**:加地图 / 加 Token / 加事件 / 加标注 / 校网格 / 建词条 / 预设揭示区域。
- **跑团现场**:切跑团态,只用揭 / 盖 / 全揭 / 切场景 / 词条聚光 / 揭示区域 / 撤销 / Checkpoint。
- **现场临时改**:进编辑态、改完、切回跑团态。但尽量别在玩家围观时改剧情卡内容。

## 跑团现场：你最常用的动作

跑团时,大部分操作其实就这 6 种:

### 揭 / 盖 / 全揭

顶栏三种主要笔刷:

| 按钮 | 作用 |
|---|---|
| 揭 | 擦开当前层迷雾 |
| 盖 | 补回当前层迷雾 |
| 全揭 | 同时擦开地形层和事件层 |

左上角切换"当前层":

| 层 | 控制什么 |
|---|---|
| 地形 | 地图图片是否可见 |
| 事件 | 事件点和部分标注是否可见 |

笔刷大小用顶栏按钮或快捷键 `1` / `2` / `3` 调整。

**为什么两层?** 因为有时候玩家走进了房间(地形该露),但你还不想剧透屋里的机关(事件该藏)。两层分开后,你可以一笔擦地形不动事件,也可以用「全揭」一笔同时擦。

### 词条聚光

跑团态点「词条」→ 选一个词条(例如"亡灵线索")→ 所有挂这个词条的事件、Token、标注会临时高亮显露。

效果是:
- 不会永久擦迷雾。
- 不改对象层级。
- 关掉词条,显示恢复正常。

如果词条还绑定了揭示区域,弹窗里会出现"揭示当前词条区域"按钮 — 这就把一键揭开变成"玩家拿到了旧地图,我按一下,对应区域就亮了"。

词条还能挂 A-Z 字母快捷键,按一下切换,再按一下关掉。

### 切场景

跑团态可以切场景(更多菜单 → 场景管理),但不显示新建 / 改名 / 删除入口。这是设计意图:跑团时只切,不改。

### 揭示区域

如果你提前预设了揭示区域,跑团时:

- 从「更多 → 揭示区域」找到区域,点「揭示」。
- 或者通过词条聚光弹窗里的"揭示当前词条区域"按钮触发。

详见「准备:揭示区域」。

**重要提醒**:跑团态区域列表会显示区域名,屏幕上玩家也能看到。所以区域命名不要剧透,用"旧地图 A""线索区域 1"这种安全名。

### 撤销 / Checkpoint

- **撤销**:`Ctrl/Cmd + Z` 或点「撤销」。能撤迷雾笔刷、全揭、揭示区域、对象创建删除等。
  - 如果一次词条揭示包含多个区域,撤销时可能要按多次。
- **Checkpoint**:重要节点前的迷雾恢复点,只回滚两层迷雾,**不**恢复 Token、事件、标注、场景结构。适合在"玩家要进 BOSS 房"前存一个。

### 跑团态 / 编辑态切换

工具默认进**跑团态**(可以放心给玩家看)。需要改东西时点顶栏「编辑」进**编辑态**(全部工具铺开)。改完点「回到跑团」收回去。

| 跑团态会暴露 | 编辑态会额外开放 |
|---|---|
| 揭 / 盖 / 全揭 / 切层 / 笔刷 / 撤销 | 加载地图、Token 上传、事件创建编辑、标注创建编辑 |
| 切场景(只切,不改) | 场景的新建 / 改名 / 删除 |
| 词条聚光 + 快捷键 | 词条的管理 |
| 揭示区域(已允许跑团态可见的) | 揭示区域创建编辑、事件快切 |
| 导入 / 导出 / 云存档 | 网格设置和校准 |
| Checkpoint | 对象面板、清理 Checkpoint |

跑团态不会显示完整对象面板,也不会显示隐藏事件列表 — 这是为了避免给玩家剧透。

### 安全选择(避免误擦)

PC 鼠标按住 `Space` 进入**安全选择层**:

- 点事件、Token、标注不会擦迷雾。
- 点空白不会误擦。
- 按住空格拖动仍可平移视图。

主要用于现场避免误操作,例如想点一下事件看内容,但担心同时擦掉那块迷雾。

## 准备阶段：每个功能怎么用

### 准备：加载第一张地图

1. 点「编辑」→「加载地图」。
2. 选 PNG / JPG / WebP 文件。
3. 大图会自动降采样到工具内可稳定操作的尺寸(单边约 4096 像素)。
4. 选初始迷雾方式:常规跑团选「全部覆盖」;只是临时演示可以不覆盖。

**不支持 HEIC**。iPad 相册里的 HEIC 建议先截图或转成 PNG / JPG。这条全程有效,后面不再重复提。

### 准备：基本视图操作

| 操作 | 结果 |
|---|---|
| 鼠标滚轮 | 缩放地图 |
| 拖动空白处 | 平移地图 |
| `Space` + 拖动 | 安全平移 |
| 双击地图 | 视图适配地图 |

如果你在现场用电脑接电视或投影,建议先把浏览器缩放恢复到 100%,再调工具内视图。

### 准备：多场景管理

编辑态打开「更多 → 场景管理」。可以新建、切换、重命名、删除。

每个场景独立保存:地图、迷雾、Token、事件、标注、词条、网格、章节信息。

跑团态也能切场景,但不暴露新建 / 改名 / 删除。

### 准备：章节信息

点底栏的章节铭牌,可编辑章节序号和章节标题。

这些信息保存在当前场景里、随场景导出 / 导入、跟随场景切换。适合长战役按章节整理地图。

### 准备：添加事件点

编辑态点「事件」,或按 `Shift` + 点地图。

每个事件可配置:

- 图标:emoji / 自定义字符 / 上传图片
- 标题、内容
- 大小
- 层级
- 是否受事件层遮罩控制
- 绑定词条

跑团态单击事件看卡片。编辑态双击事件编辑。

事件适合:机关、陷阱、遭遇、NPC 位置、线索、DM 现场提示。

### 准备：添加 Token

编辑态点「Token」,上传图片。

Token 可配置:名称、形状(圆 / 方 / 原图比例)、大小、阵营描边、绑定词条。

适合:玩家、敌人、NPC、载具、临时战术标记。

### 准备：添加文字标注

编辑态点「文字」。

文字标注可配:文本、字号、颜色、大小、位置、遮罩归属、绑定词条。

**遮罩归属**有三种:

- **公开**:始终显示。
- **受地形遮罩**:地形揭开后显示。
- **受事件遮罩**:事件层揭开后显示。

### 准备：添加图片标注

编辑态点「图片」,上传 PNG / JPG / WebP。

适合:地图徽记、阵营图标、小型说明图、临时路线图、战术标记。

跟文字标注一样,可设遮罩归属和绑定词条。

### 准备：词条

编辑态点「词条 → 管理词条」。

每个词条可配:名称、颜色、是否允许跑团态出现、A-Z 快捷键。

事件、Token、文字标注、图片标注、揭示区域都可绑定词条。

跑团态使用见「跑团现场:词条聚光」。

### 准备：揭示区域

**这是 v3.20 新增的大功能**,用于"玩家获得某张旧地图或某条情报后,某片地图直接亮起"。

操作步骤:

1. 进入编辑态。
2. 确保已加载地图并开启网格(揭示区域基于网格)。
3. 「更多 → 揭示区域」→「新建区域」。
4. 输入区域名称(注意:跑团态会看到这个名,别剧透)。
5. 选揭示目标:地形层 / 事件层 / 双层。
6. 选要绑定的词条。
7. 点「开始选格」。
8. 在地图上拖动选格;按住 `Alt` 拖动可以从区域里移除格。
9. 点底部「保存区域」。

区域会按当前网格保存,**并保存网格快照**。即使之后重新校准当前网格,旧区域也不会跟着漂移。

**它不是什么**:

- 不是自动触发器(它不会因为某个 Token 走进区域就自动揭开)。
- 不是每格属性数据库(它不存"这格有陷阱"之类元数据)。
- 它只是 DM 手动执行的预设区域工具。

跑团态使用见「跑团现场:揭示区域」。

### 准备：网格设置

编辑态打开网格设置。

可选类型:六边形 / 方形。

可调:格子大小、偏移、旋转、线条颜色、线条透明度。

用途:让迷雾按整格揭示、对齐原图自带网格、作为揭示区域的基础。

### 准备：网格校准

如果原图自带网格,可以用校准面板对齐。

常用流程:

1. 打开网格,选六边形或方形。
2. 调整格子大小让它跟原图格大致一致。
3. 拖动网格偏移让线条落到原图线条上。
4. 需要时调旋转。
5. 确认保存。

校准过程中,工具会拦截其他编辑操作,避免半成品网格进入存档。

## 进阶功能

### 进阶：使用对象面板

编辑态右侧或窄屏底部会出现对象面板。

可以:

- 列出全部事件、Token、文字标注、图片标注
- 按类型筛选
- 搜索名称、内容、词条
- 选中对象
- 定位到对象附近
- 打开对象编辑弹窗

当对象很多时,这比在地图上逐个找更可靠。

### 进阶：事件快切

编辑态打开「更多 → 事件快切」。

事件快切用于快速搜索事件,并把事件切到:

- 显露在事件遮罩上方
- 放回事件遮罩下方

适合现场临时让某个事件点出现或隐藏。跑团态不显示这个入口,避免隐藏事件列表泄露。

### 进阶：词条快捷键

词条可绑定 A-Z 字母快捷键。跑团中按对应字母:

- 第一次按:切到该词条聚光。
- 再按一次:关闭该词条聚光。
- 按另一个词条快捷键:切到另一个词条。

输入框打字时不会触发词条快捷键。

### 进阶：Checkpoint

Checkpoint 是迷雾恢复点,主要用于现场安全回滚。

它**恢复**:地形层迷雾、事件层迷雾。

它**不恢复**:Token、事件、标注、场景结构、地图图片。

适合在重要节点前保存一次,例如玩家进入大房间、战斗开始前、揭示大区域前。

## 跨设备：备份与恢复

### 跨设备：本机自动保存

浏览器会把当前项目自动保存到本机 IndexedDB。只要不清理这个网站的浏览器数据,下次打开会自动恢复。

清浏览器站点数据 = 删本机保存。所以重要进度记得导出或上传云存档。

### 跨设备：导出全部场景

「更多 → 导出全部场景」,得到 `.fogbundle.json`。

它**包含**:

- 全部场景
- 场景名称和章节信息
- 地图尺寸和地图指纹
- 两层迷雾
- 事件、Token、文字标注、图片标注
- 词条和快捷键
- 揭示区域
- 网格、当前视图、设置

它**不包含**地图原图。这样文件更小,也避免把购买的地图素材直接打包。

### 跨设备：导入全部场景

「更多 → 导入全部场景」,选 `.fogbundle.json`。

工具会要求你为每个场景重新选择本机地图,**地图尺寸需要匹配**。如果地图缺失或过大,可以走「占位地图恢复向导」(下一节)。

如果导入包里记录了原"当前场景",导入后会优先打开它。

### 跨设备：云存档

「更多 → 云存档」。

云存档用你自己的私有 GitHub 仓库存进度,可跨电脑同步。

基本流程:

1. 准备一把 GitHub fine-grained PAT(私有访问钥匙,GitHub 网页生成)。
2. 在工具内输入云存档钥匙。
3. 连接私有仓库。
4. 点「上传当前进度」。
5. 到另一台电脑打开同一网页。
6. 输入读取钥匙。
7. 点「读取云端进度」。

云存档**会上传**:状态数据、普通地图图片、能优化到阈值内的大地图图片。

对**真正太大的地图**(超过阈值的高分辨率原图),云存档只保存:地图尺寸、地图指纹、迷雾、Token、事件、标注、词条、揭示区域。换电脑读取后让你重新绑定同尺寸地图,这就是下一节「占位地图」的场景。

### 跨设备：绑定占位地图

如果读取云存档后某个场景显示占位地图,说明当前电脑没有对应大图本体。

编辑态打开占位地图恢复向导,可以:

- 绑定本机图。
- 绑定云端地图库图片。

绑定时**必须尺寸匹配**。尺寸不匹配时工具会拒绝绑定,避免遮罩和对象错位。

### 跨设备：云端地图库(可选)

如果你把大图手动放进私有仓的 `manual-maps/` 目录,工具可以在占位地图恢复时直接从云端选图。

可选的 `manual-maps/index.json` 可以提前提供:图片路径、宽高、文件大小、备注。这样可以避免下载几十 MB 后才发现尺寸不对。最终绑定时,工具仍会真正下载图片并校验真实尺寸。

## 数据与隐私

- 本地进度保存在当前浏览器的 IndexedDB。
- GitHub Pages 只提供静态网页。
- 工具没有项目作者维护的服务器。
- 不使用账号系统。
- 不会把地图自动上传到公开仓库。
- `.fogbundle.json` 不包含地图原图。
- 如果使用云存档,状态和普通地图会上传到**你自己控制的**私有 GitHub 仓库。
- 清理浏览器站点数据会删除本机保存的项目。

推荐:Chrome、Edge、Firefox、Safari、iPad Safari。

iPhone 小屏可以打开,但不是主要优化目标。

## 离线使用

如果不想联网,可以下载仓库里的 `index.html`,用浏览器直接打开。

离线时:

- 主功能仍可用。
- 本机保存仍可用。
- 导出 / 导入仍可用。
- **云存档不可用**(因为需要访问 GitHub API)。

---

## 当前限制

- 不导出 PNG 成图,需要图片时请用系统截图。
- 不做多人实时同步。
- 不做玩家权限模式。
- 不做自动视野、触发器、传送门等电子游戏逻辑。
- 单张地图最大边长约 4096 像素,超过会自动降采样。
- 标注按对象中心点判断是否被遮罩,不做逐像素裁切。
- 一个词条如果绑定多个揭示区域,一次揭示会产生多个撤销项,撤销时可能要按多次。
- 跑团态的揭示区域列表会显示区域名和格数,区域命名应避免剧透。

---

## 技术特点

- 单文件 `index.html`
- 无后端
- 无构建步骤
- 无 npm 依赖
- 无 CDN 依赖
- 可直接部署到 GitHub Pages
- 主要数据保存在浏览器 IndexedDB
- 支持导出 JSON 场景包
