# 战争迷雾地图工具

一个给 DND / TRPG 跑团使用的网页地图工具：在同一块屏幕上展示地图，逐步揭示战争迷雾，管理事件提示、Token、六边形 / 方形网格、临时地图标注和信息词条聚光。

**在线使用：**  
[https://martinsin0223.github.io/DNDtool/](https://martinsin0223.github.io/DNDtool/)

## 这是什么

这是一个面向桌面跑团现场的共享地图工具。DM 可以把地图投到玩家能看到的屏幕上，一边跑团一边擦开迷雾、移动 Token、展示地点标注或事件提示。

它不是 VTT，不做规则自动化，也不做玩家账号、后台权限、视野算法、触发器或实时联机。它解决的是一个更朴素的问题：让 DM 在现场快速、稳定、可控地展示一张带迷雾的地图。

## 适合谁

- 线下跑团的 DM
- 用电视、投影、平板或共享电脑展示地图的桌面团
- 想要“比截图强，但比完整 VTT 轻”的地图工具
- 需要临时文字、图片、事件点、Token 和整格遮罩的战役准备者

## 核心能力

### 跑团态与编辑态

工具默认进入跑团态，屏幕适合长期给玩家看：保留揭示、覆盖、全揭、切场景、查看事件、导入和导出。

需要准备或临时修改时，点击顶栏「编辑」进入编辑态。编辑态会展开地图上传、事件创建、Token、文字、图片、网格设置、场景管理等完整工具，并在侧边显示「对象」面板。

PC 鼠标操作中，按住空格会进入安全选择层：点击事件、Token 或标注不会擦除迷雾；拖动超过阈值仍会平移地图。

「对象」面板会列出当前场景里的事件、Token、文字标注和图片标注。可以按类型筛选、按名称或内容搜索；点击可选中地图对象，定位按钮会把视图移到对象附近，编辑按钮会直接打开对应编辑窗口，适合快速找到很小的文字或图片。

v3.8 起，界面采用黑曜石暗底、黄铜控制台按钮和任务板底栏的纯 CSS 视觉系统；没有外部图片资源，地图仍然是主视觉中心。

底栏章节信息可以为每个场景单独编辑：章节序号和章节标题会保存在本机，并随“导出全部场景”一起带走。

### 词条聚光

事件、Token、文字标注和图片标注都可以挂载一个或多个“词条”。编辑态负责创建、重命名、删除词条，并决定哪些词条允许出现在跑团态选择器里。

跑团时点击顶栏「词条」，选择一个现场词条后，所有挂载该词条的对象会临时高亮；被事件雾遮住的事件图标、受地形/事件遮罩控制的标注也会临时显露。清除词条后，一切恢复原本遮罩状态，不会擦掉迷雾，也不会把事件永久改到遮罩上层。

### 双层战争迷雾

工具有两层独立遮罩：

| 层 | 用途 |
|---|---|
| 地形层 | 遮住地图本身，控制玩家是否看见地形 |
| 事件层 | 遮住事件点和部分标注，控制玩家是否看见提示信息 |

笔刷支持：

| 模式 | 作用 |
|---|---|
| 揭 | 擦除当前层遮罩 |
| 盖 | 补回当前层遮罩 |
| 全揭 | 同时擦除地形层和事件层遮罩 |

### 事件点

事件点适合放机关、遭遇、线索、地点说明或 DM 现场提示。

- 支持默认 emoji、自定义字符、上传图片作为图标
- 每个事件有标题和内容
- 跑团态可单击查看；编辑态可双击编辑、移动、删除
- 可设置为永远可见，或受事件层遮罩控制

### 地图标注

标注是直接放在地图上的文字或图片，适合地名、路线、阵营标识、说明图、临时战术图标。

| 类型 | 能力 |
|---|---|
| 文字标注 | 自定义文字、字号、颜色、移动、缩放 |
| 图片标注 | 上传 PNG / JPG / WebP，移动、缩放 |

标注可以选择：

- 公开：始终显示
- 受地形遮罩：地形层揭开后显示
- 受事件遮罩：事件层揭开后显示

### Token

Token 用来表示玩家、敌人、NPC 或其他棋子。

- 支持上传图片
- 支持圆形、方形、原图比例
- 支持阵营描边
- 可拖动、缩放、编辑、删除

### 网格遮罩

可以打开六边形或方形网格，让笔刷按整格揭示或覆盖。

- 支持网格大小调整
- 支持平移校准
- 支持旋转校准
- 可对准地图原图自带的 hex 或方格网格

## 基本使用

1. 打开在线地址。
2. 点击「编辑」，再点击「加载地图」，加载 PNG / JPG / WebP 地图。
3. 初始迷雾通常选择「全部覆盖」。
4. 准备事件、Token、文字、图片或网格。
5. 点击「回到跑团」，现场用笔刷逐步揭示地图。
6. 需要同时露出地形和事件时，使用「全揭」。
7. 需要跨设备或备份时，使用「导出全部场景」。

## 导出与导入

工具可以把全部场景导出为 `.fogbundle.json`。

导出内容包括：

- 全部场景、场景名称、当前场景标记、地图指纹和地图尺寸
- 两层迷雾：地形层和事件层
- Token：图片、形状、描边、名称、位置、大小、层级
- 事件点：emoji / 自定义图标 / 上传图片、标题、内容、位置、大小、层级、是否受事件雾遮罩、事件快切、词条挂载
- 地图标注：文字、图片、位置、大小、颜色、字号、层级、遮罩层归属、词条挂载
- 词条：场景词条列表、A-Z 快捷键、跑团态可见设置、当前聚光词条、对象挂载关系
- 网格、笔刷、迷雾透明度、迷雾颜色、当前视图
- 自定义主题配置

导出内容不包括地图原图，也不包括本机临时 Checkpoint。导入时会进入场景导入向导，为每个场景选择本机地图，工具会检查地图尺寸；如果导入包里的原当前场景也被导入，工具会优先打开它。这样做可以减少文件体积，也避免把地图素材直接打包进导出文件。

旧版 `.fogsave.json` 单场景文件仍可导入，但主流程已经改为全部场景导出。

## 云存档

工具现在支持把整套跑团进度存进你自己的私有 GitHub 仓库，再在另一台电脑上直接读回来。

云存档包含：

- 全部场景状态
- 两层迷雾
- Token、事件、文字标注、图片标注、词条聚光关系
- 当前视图和网格设置
- 地图图片（普通图自动上传；能优化成小文件的大尺寸图也会上传；真正过大的图可只保存占位几何和进度，另一台电脑读取后绑定本机原图或私有仓 `manual-maps/` 里的云端地图库图片）

云存档不走这个公开仓库，也不会传到项目作者服务器。它使用的是你自己控制的私有仓库 `DNDtool-state`。

### 云存档能做什么

- 连接 GitHub fine-grained PAT
- 上传当前全部进度
- 在另一台电脑读取最新进度
- 查看历史快照并恢复
- 断网或中断后继续未完成上传
- 对重复地图按内容去重，同一张图只存一份
- 对大状态和普通地图自动小分片上传；能压到阈值内的优化地图继续上传，超过阈值的地图才跳过图片本体，只同步遮罩、Token、事件、标注和地图尺寸
- 读取到占位地图后，可以重新选择同尺寸本机原图，或在编辑态从云端地图库绑定 `manual-maps/` 里的同尺寸图片，遮罩和对象会自动对回去
- 云端地图库可选支持 `manual-maps/index.json`，提前显示大图尺寸、文件大小和备注；这些信息只用于预判，真正绑定时仍会下载图片并校验实际尺寸
- 编辑态事件快切可以搜索事件并一键把事件显露到遮罩上层，或放回事件遮罩下；跑团态不暴露隐藏事件列表
- 手动清理云端孤儿地图和老快照

### 云存档的边界

- 不是多人实时协作
- 不是账号系统
- 不会自动后台同步
- 朋友电脑如果只给读取钥匙，可以只读不能上传

## 数据与隐私

- 地图和进度保存在浏览器本机 IndexedDB。
- 工具没有后端，不会把地图上传到服务器。
- GitHub Pages 只负责提供这个静态网页。
- 导出的 `.fogbundle.json` 不包含地图原图。
- 如果你使用云存档，普通地图图片和跑团状态会上传到你自己的私有 GitHub 仓库；超大地图默认不上传图片本体，只保存占位几何和跑团进度。
- 清理浏览器站点数据会删除本机保存的场景。

## 快捷键

| 快捷键 | 功能 |
|---|---|
| `E` | 揭示当前层 |
| `R` | 覆盖当前层 |
| `Q` | 全揭两层 |
| `1` / `2` / `3` | 笔刷小 / 中 / 大 |
| `F` | 演示模式 |
| `Space` + 拖动 | 平移视图 |
| `Shift` + 点击地图 | 添加事件（编辑态） |
| `Ctrl/Cmd + Z` | 撤销 |
| `Delete` / `Backspace` | 删除选中对象（编辑态） |
| `Esc` | 取消选中或关闭弹窗 |

## 浏览器支持

推荐使用最新版 Chrome、Edge、Firefox、Safari。

iPad Safari 可用。iPhone 小屏可以打开，但不是主要优化目标。

不支持 HEIC 图片。iPad 相册里的 HEIC 建议先截图或转成 PNG / JPG。

## 技术特点

- 单文件 `index.html`
- 无后端
- 无构建步骤
- 无 npm 依赖
- 无 CDN 依赖
- 可直接部署到 GitHub Pages

## 离线使用

如果不想联网，也可以下载仓库里的 `index.html`，用浏览器打开。离线使用时数据同样保存在当前浏览器本机。

## 当前限制

- 不导出 PNG 成图，需要图片时请用系统截图。
- 不做多人实时同步。
- 不做玩家权限模式。
- 不做自动视野、触发器、传送门等电子游戏逻辑。
- 单张地图最大边长约 4096 像素，超过会自动降采样。
- 标注按对象中心点判断是否被遮罩，不做逐像素裁切。
