# 战争迷雾地图工具

一个给 DND / TRPG 跑团使用的网页地图工具：在同一块屏幕上展示地图，逐步揭示战争迷雾，管理事件提示、Token、六边形 / 方形网格和临时地图标注。

**在线使用：**  
[https://martinsin0223.github.io/DNDtool/](https://martinsin0223.github.io/DNDtool/)

## 这是什么

这是一个面向桌面跑团现场的共享地图工具。DM 可以把地图投到玩家能看到的屏幕上，一边跑团一边擦开迷雾、移动 Token、展示地点标注或事件提示。

它不是 VTT，不做规则自动化，也不做玩家账号、后台权限、视野算法、触发器或实时联机。它解决的是一个更朴素的问题：让 DM 在现场快速、稳定、可控地展示一张带迷雾的地图。

## 适合谁

- 线下跑团的 DM
- 用电视、投影、平板或共享电脑展示地图的桌面团
- 想要“比截图强，但比完整 VTT 轻”的地图工具
- 需要临时文字、图片、事件点、Token 和整格遮罩的战役准备者

## 核心能力

### 跑团态与编辑态

工具默认进入跑团态，屏幕适合长期给玩家看：保留揭示、覆盖、全揭、切场景、查看事件、导入和导出。

需要准备或临时修改时，点击顶栏「编辑」进入编辑态。编辑态会展开地图上传、事件创建、Token、文字、图片、网格设置、场景管理等完整工具，并在侧边显示「对象」面板。

「对象」面板会列出当前场景里的事件、Token、文字标注和图片标注。可以按类型筛选、按名称或内容搜索；点击可选中地图对象，定位按钮会把视图移到对象附近，编辑按钮会直接打开对应编辑窗口，适合快速找到很小的文字或图片。

### 双层战争迷雾

工具有两层独立遮罩：

| 层 | 用途 |
|---|---|
| 地形层 | 遮住地图本身，控制玩家是否看见地形 |
| 事件层 | 遮住事件点和部分标注，控制玩家是否看见提示信息 |

笔刷支持：

| 模式 | 作用 |
|---|---|
| 揭 | 擦除当前层遮罩 |
| 盖 | 补回当前层遮罩 |
| 全揭 | 同时擦除地形层和事件层遮罩 |

### 事件点

事件点适合放机关、遭遇、线索、地点说明或 DM 现场提示。

- 支持默认 emoji、自定义字符、上传图片作为图标
- 每个事件有标题和内容
- 跑团态可单击查看；编辑态可双击编辑、移动、删除
- 可设置为永远可见，或受事件层遮罩控制

### 地图标注

标注是直接放在地图上的文字或图片，适合地名、路线、阵营标识、说明图、临时战术图标。

| 类型 | 能力 |
|---|---|
| 文字标注 | 自定义文字、字号、颜色、移动、缩放 |
| 图片标注 | 上传 PNG / JPG / WebP，移动、缩放 |

标注可以选择：

- 公开：始终显示
- 受地形遮罩：地形层揭开后显示
- 受事件遮罩：事件层揭开后显示

### Token

Token 用来表示玩家、敌人、NPC 或其他棋子。

- 支持上传图片
- 支持圆形、方形、原图比例
- 支持阵营描边
- 可拖动、缩放、编辑、删除

### 网格遮罩

可以打开六边形或方形网格，让笔刷按整格揭示或覆盖。

- 支持网格大小调整
- 支持平移校准
- 支持旋转校准
- 可对准地图原图自带的 hex 或方格网格

## 基本使用

1. 打开在线地址。
2. 点击「编辑」，再点击「加载地图」，加载 PNG / JPG / WebP 地图。
3. 初始迷雾通常选择「全部覆盖」。
4. 准备事件、Token、文字、图片或网格。
5. 点击「回到跑团」，现场用笔刷逐步揭示地图。
6. 需要同时露出地形和事件时，使用「全揭」。
7. 需要跨设备或备份时，使用「导出全部场景」。

## 导出与导入

工具可以把全部场景导出为 `.fogbundle.json`。

导出内容包括：

- 全部场景、场景名称、当前场景标记、地图指纹和地图尺寸
- 两层迷雾：地形层和事件层
- Token：图片、形状、描边、名称、位置、大小、层级
- 事件点：emoji / 自定义图标 / 上传图片、标题、内容、位置、大小、层级、是否受事件雾遮罩
- 地图标注：文字、图片、位置、大小、颜色、字号、层级、遮罩层归属
- 网格、笔刷、迷雾透明度、迷雾颜色、当前视图
- 自定义主题配置

导出内容不包括地图原图，也不包括本机临时 Checkpoint。导入时会进入场景导入向导，为每个场景选择本机地图，工具会检查地图尺寸；如果导入包里的原当前场景也被导入，工具会优先打开它。这样做可以减少文件体积，也避免把地图素材直接打包进导出文件。

旧版 `.fogsave.json` 单场景文件仍可导入，但主流程已经改为全部场景导出。

## 数据与隐私

- 地图和进度保存在浏览器本机 IndexedDB。
- 工具没有后端，不会把地图上传到服务器。
- GitHub Pages 只负责提供这个静态网页。
- 导出的 `.fogbundle.json` 不包含地图原图。
- 清理浏览器站点数据会删除本机保存的场景。

## 快捷键

| 快捷键 | 功能 |
|---|---|
| `E` | 揭示当前层 |
| `R` | 覆盖当前层 |
| `Q` | 全揭两层 |
| `1` / `2` / `3` | 笔刷小 / 中 / 大 |
| `F` | 演示模式 |
| `Space` + 拖动 | 平移视图 |
| `Shift` + 点击地图 | 添加事件（编辑态） |
| `Ctrl/Cmd + Z` | 撤销 |
| `Delete` / `Backspace` | 删除选中对象（编辑态） |
| `Esc` | 取消选中或关闭弹窗 |

## 浏览器支持

推荐使用最新版 Chrome、Edge、Firefox、Safari。

iPad Safari 可用。iPhone 小屏可以打开，但不是主要优化目标。

不支持 HEIC 图片。iPad 相册里的 HEIC 建议先截图或转成 PNG / JPG。

## 技术特点

- 单文件 `index.html`
- 无后端
- 无构建步骤
- 无 npm 依赖
- 无 CDN 依赖
- 可直接部署到 GitHub Pages

## 离线使用

如果不想联网，也可以下载仓库里的 `index.html`，用浏览器打开。离线使用时数据同样保存在当前浏览器本机。

## 当前限制

- 不导出 PNG 成图，需要图片时请用系统截图。
- 不做多人实时同步。
- 不做玩家权限模式。
- 不做自动视野、触发器、传送门等电子游戏逻辑。
- 单张地图最大边长约 4096 像素，超过会自动降采样。
- 标注按对象中心点判断是否被遮罩，不做逐像素裁切。
