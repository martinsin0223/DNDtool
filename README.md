# 战争迷雾地图工具

一个给 DND / TRPG 跑团使用的单文件网页地图工具。它用于在同一块共享屏幕上展示地图、逐步揭示战争迷雾、管理事件提示、Token、地图标注、信息词条和预设揭示区域。

**在线使用：**  
[https://martinsin0223.github.io/DNDtool/](https://martinsin0223.github.io/DNDtool/)

当前版本：**v3.20.1**

---

## 项目愿景

这个工具的目标不是替代完整 VTT，也不是做规则自动化平台。它要解决的是线下跑团里最常见、也最容易被复杂软件拖慢的问题：

> DM 有一张地图，玩家看着同一块屏幕，DM 需要稳定、快速、可控地揭开他们已经探索或获得情报的区域。

所以它坚持几个产品原则：

- **地图优先**：地图本身永远是屏幕主角，UI 只服务于现场操作。
- **DM 手动控制**：不做自动视野、触发器、传送门、规则判定。
- **共享屏幕可信**：跑团态默认克制，不暴露编辑面板和隐藏对象列表。
- **本地优先**：不需要登录，不需要服务器，网页打开就能用。
- **静态部署**：公开版只有 `README.md` 和 `index.html`，可以放在 GitHub Pages 这类静态托管上。
- **可搬家**：本机保存、导出场景包、云存档三条路径并存，尽量降低现场换电脑时的风险。

它更像一张带工具层的电子战斗地图，而不是游戏引擎。

## 它适合谁

- 线下跑团的 DM
- 用电视、投影、平板或共享电脑展示地图的桌面团
- 想要比截图灵活、但比完整 VTT 轻很多的地图展示工具
- 需要提前准备多场景、事件点、Token、文字标注、图片标注和遮罩的战役主持人
- 需要在现场根据玩家获得的信息，快速揭开某些预设地图区域的 DM

## 它不做什么

- 不做玩家账号。
- 不做 DM / 玩家权限分离。
- 不做多人实时同步。
- 不做自动视野。
- 不做规则自动化。
- 不导出 PNG 成图。
- 不依赖后端、React、npm、CDN 或外部资源。

这些不是暂时没做，而是当前主项目的产品边界。

## 快速开始

1. 打开在线地址：  
   [https://martinsin0223.github.io/DNDtool/](https://martinsin0223.github.io/DNDtool/)
2. 点击顶栏「编辑」进入编辑态。
3. 点击「加载地图」，选择 PNG / JPG / WebP 地图。
4. 初始迷雾通常选择「全部覆盖」。
5. 按需要添加事件、Token、文字标注、图片标注、网格和词条。
6. 点击「回到跑团」，把屏幕交给玩家看。
7. 用「揭」「盖」「全揭」逐步控制地图可见区域。
8. 跑团结束或换电脑前，使用「导出全部场景」或「云存档」保存进度。

浏览器会把当前项目保存在本机 IndexedDB。只要不清理这个网站的浏览器数据，下次打开会自动恢复。

## 界面模式

### 跑团态

跑团态是默认模式，适合长期展示给玩家看。

跑团态保留现场常用操作：

- 揭示 / 覆盖 / 全揭迷雾
- 切换地形层和事件层
- 调整笔刷大小
- 撤销
- 查看词条聚光
- 查看已允许跑团态使用的揭示区域
- 切换场景
- 导入 / 导出
- 打开云存档

跑团态不会显示完整对象面板，也不会显示隐藏事件列表。

### 编辑态

点击顶栏「编辑」进入编辑态。编辑态用于准备地图或现场临时改图。

编辑态会开放：

- 加载 / 替换地图
- 新建 / 编辑 / 删除事件
- 上传 / 编辑 / 删除 Token
- 添加文字标注
- 添加图片标注
- 设置六边形或方形网格
- 校准网格位置、大小和旋转
- 管理词条
- 创建和编辑揭示区域
- 管理场景
- 查看对象面板
- 使用事件快切
- 清理 Checkpoint

编辑态适合 DM 操作，不适合长期展示给玩家。

### 安全选择

PC 鼠标操作中，按住 `Space` 可以进入安全选择层。

用途：

- 点击事件、Token、标注时不会擦除迷雾。
- 点击空白处不会误擦迷雾。
- 按住空格拖动仍可以平移地图。

这主要用于现场避免误操作。

## 核心概念

### 地形层

地形层遮住地图图片本身。玩家是否看见房间、道路、山谷、区域形状，主要由地形层控制。

### 事件层

事件层遮住事件点和部分标注。它用于控制玩家是否看见线索、机关、地点名、图标或 DM 提示。

### 全揭

「全揭」会同时擦除地形层和事件层。适合玩家进入一个区域后，地形和事件提示都应该同时显示的场景。

### 跑团态 / 编辑态

跑团态是玩家可长期观看的现场界面。编辑态是 DM 准备和维护界面。

### 词条

词条是信息标签。事件、Token、文字标注、图片标注和揭示区域都可以绑定词条。

例子：

- “藏宝图”
- “亡灵线索”
- “城堡二层”
- “玩家已获得的旧地图”

跑团中选择一个词条后，绑定该词条的对象会被临时显露或高亮。

### 揭示区域

揭示区域是 v3.20 新增的大功能。DM 可以在编辑态按网格预先圈出一组格子，给它命名，并绑定词条。跑团时玩家获得地图或情报后，DM 可以一键揭开对应区域。

它不是自动触发器，也不是每格属性数据库。它只是 DM 手动执行的预设区域工具。

| 功能 | 说明 |
|---|---|
| 双层迷雾 | 地形层和事件层分离控制 |

| Token | 放置玩家、敌人、NPC 或棋子 |

| 图片标注 | 在地图上放小图、徽记、战术图标 |
| 词条聚光 | 临时显露 / 高亮同词条对象 |
| 词条快捷键 | A-Z 按键快速切换现场词条 |
| 揭示区域 | 预设网格区域，跑团时一键揭开 |
| 网格遮罩 | 六边形 / 方形网格，按整格揭示或覆盖 |
| 网格校准 | 调整网格大小、位置、旋转 |
| 对象面板 | 编辑态搜索、筛选、定位、编辑对象 |
| 事件快切 | 编辑态快速搜索事件并切换事件层显示状态 |
| Checkpoint | 保存迷雾恢复点，便于现场回滚两层遮罩 |
| 章节信息 | 每个场景可编辑章节序号和标题 |

## 详细操作手册

### 1. 加载第一张地图

1. 点击「编辑」。
2. 点击「加载地图」。
3. 选择 PNG / JPG / WebP 文件。
4. 如果地图很大，工具会自动降采样到可稳定操作的尺寸。
5. 选择初始迷雾方式。

建议：

- 常规跑团选择“全部覆盖”。
- 如果只是临时演示，可以选择不覆盖或部分覆盖后再手动补。

不支持 HEIC。iPad 相册里的 HEIC 建议先截图或转成 PNG / JPG。

### 2. 基本视图操作

| 操作 | 结果 |
|---|---|
| 鼠标滚轮 | 缩放地图 |
| 拖动空白处 | 平移地图 |
| `Space` + 拖动 | 安全平移 |
| 双击地图 | 视图适配地图 |

如果你在现场用电脑连电视或投影，建议先把浏览器缩放恢复到 100%，再调整工具内地图视图。

### 3. 揭示和覆盖迷雾

顶栏有三种主要笔刷：

| 按钮 | 作用 |
|---|---|
| 揭 | 擦开当前层迷雾 |
| 全揭 | 同时擦开地形层和事件层 |
| 盖 | 补回当前层迷雾 |

左上角可以切换当前层：

| 层 | 作用 |
|---|---|
| 地形 | 控制地图图片是否可见 |
| 事件 | 控制事件点和部分标注是否可见 |

笔刷大小可用按钮或快捷键调整。

### 4. 撤销

点击「撤销」或按 `Ctrl/Cmd + Z` 可以撤销最近操作。

可撤销的典型操作：

- 迷雾笔刷
- 全揭笔刷
- 揭示区域
- 创建 / 删除 / 修改部分对象

注意：如果一次词条揭示包含多个揭示区域，当前版本会按区域生成多个撤销项，撤销时可能需要按多次。

### 5. 创建和管理场景

编辑态打开「更多 -> 场景管理」。

你可以：

- 新建场景
- 切换场景
- 重命名场景
- 删除场景
- 给不同场景加载不同地图
- 为每个场景保存独立迷雾、Token、事件、标注、词条、网格和章节信息

跑团态也可以切换场景，但不会暴露完整编辑入口。

### 6. 编辑章节信息

点击底栏章节铭牌，可以编辑当前场景的章节序号和标题。

这些信息会：

- 保存在本机
- 随场景导出 / 导入
- 跟随场景切换

适合长战役按章节整理地图。

### 7. 添加事件点

编辑态点击「事件」，或按住 `Shift` 点击地图添加事件。

事件可以配置：

- 图标：emoji、自定义字符、上传图片
- 标题
- 内容
- 大小
- 层级
- 是否受事件层遮罩控制
- 绑定词条

跑团态单击可查看事件卡片。编辑态双击可编辑事件。

事件适合：

- 机关
- 陷阱
- 遭遇
- NPC 位置
- 线索
- DM 现场提示

### 8. 使用事件快切

编辑态打开「更多 -> 事件快切」。

事件快切用于快速搜索事件，并把事件切到：

- 显露在事件遮罩上方
- 放回事件遮罩下方

它适合现场临时让某个事件点出现或隐藏。跑团态不会显示这个入口，避免隐藏事件列表泄露。

### 9. 添加 Token

编辑态点击「Token」，上传图片。

Token 可以配置：

- 名称
- 圆形 / 方形 / 原图比例
- 大小
- 阵营描边
- 绑定词条

Token 可拖动、编辑、删除。适合玩家、敌人、NPC、载具或临时战术标记。

### 10. 添加文字标注

编辑态点击「文字」。

文字标注可以配置：

- 文本
- 字号
- 颜色
- 大小
- 位置
- 遮罩归属
- 绑定词条

遮罩归属包括：

- 公开：始终显示
- 受地形遮罩：地形揭开后显示
- 受事件遮罩：事件层揭开后显示

### 11. 添加图片标注

编辑态点击「图片」，上传 PNG / JPG / WebP。

图片标注适合：

- 地图上的徽记
- 阵营图标
- 小型说明图
- 临时路线图
- 战术标记

图片标注同样可以设置遮罩归属和绑定词条。

### 12. 使用对象面板

编辑态右侧或窄屏底部会出现对象面板。

对象面板可以：

- 列出事件、Token、文字标注、图片标注
- 按类型筛选
- 搜索名称、内容、词条
- 选中对象
- 定位到对象附近
- 打开对象编辑弹窗

当地图对象很多时，对象面板比在地图上逐个找更可靠。

### 13. 创建词条

编辑态点击「词条 -> 管理词条」。

词条可以配置：

- 名称
- 颜色
- 是否允许跑团态出现
- A-Z 快捷键

事件、Token、文字标注、图片标注和揭示区域都可以绑定词条。

### 14. 使用词条聚光

跑团态点击「词条」，选择一个词条。

效果：

- 绑定该词条的事件、Token、标注会临时高亮或显露。
- 不会永久擦除迷雾。
- 不会改变对象层级。
- 清除词条后恢复正常显示。

如果词条绑定了揭示区域，弹窗里会出现“揭示当前词条区域”按钮。点击后会永久揭开这些预设区域。

### 15. 使用词条快捷键

词条可以绑定 A-Z 的英文字母快捷键。

跑团中按对应字母：

- 第一次按：切到该词条聚光
- 再按一次：关闭该词条聚光
- 按另一个词条快捷键：切换到另一个词条

在输入框里打字时不会触发词条快捷键。

### 16. 创建揭示区域

揭示区域是 v3.20 新增功能，用于“玩家获得某张地图或某条情报后，某片地图直接亮起”。

操作步骤：

1. 进入编辑态。
2. 确保已经加载地图并开启网格。
3. 打开「更多 -> 揭示区域」。
4. 点击「新建区域」。
5. 输入区域名称。
6. 选择揭示目标：地形层、事件层或双层。
7. 选择要绑定的词条。
8. 点击「开始选格」。
9. 在地图上拖动选择格子。
10. 按住 `Alt` 拖动可以从区域里移除格子。
11. 点击底部面板「保存区域」。

区域会按当前网格保存，并保存网格快照。即使之后重新校准当前网格，旧区域也不会跟着漂移。

### 17. 跑团中揭示区域

方式一：从区域列表揭示。

1. 跑团态打开「更多 -> 揭示区域」。
2. 找到区域。
3. 点击「揭示」。

方式二：从词条聚光揭示。

1. 跑团态点击「词条」。
2. 选择对应词条。
3. 如果该词条绑定了区域，点击“揭示当前词条区域”。

建议区域名称不要剧透。跑团态区域列表会显示区域名和格数，如果屏幕正在给玩家看，区域名最好写成“旧地图 A”“线索区域 1”这类安全名称。

### 18. 设置网格

编辑态打开网格设置。

可选类型：

- 六边形
- 方形

可调整：

- 格子大小
- 偏移位置
- 旋转角度
- 线条颜色
- 线条透明度

网格用途：

- 让迷雾按整格揭示
- 对齐原图自带网格
- 创建揭示区域

### 19. 校准网格

如果原图自带网格，可以用校准面板对齐。

常用流程：

1. 打开网格。
2. 选择六边形或方形。
3. 调整格子大小。
4. 拖动网格偏移。
5. 需要时调整旋转。
6. 确认保存。

校准过程中，工具会拦截其他编辑操作，避免半成品网格进入存档。

### 20. 使用 Checkpoint

Checkpoint 是迷雾恢复点，主要用于现场安全回滚。

它恢复的是：

- 地形层迷雾
- 事件层迷雾

它不恢复：

- Token
- 事件
- 标注
- 场景结构
- 地图图片

适合在重要节点前保存一次，比如玩家进入大房间、战斗开始前、揭示大区域前。

### 21. 导出全部场景

打开「更多 -> 导出全部场景」。

导出文件是 `.fogbundle.json`。

它包含：

- 全部场景
- 场景名称和章节信息
- 地图尺寸和地图指纹
- 两层迷雾
- 事件
- Token
- 文字标注
- 图片标注
- 词条和快捷键
- 揭示区域
- 网格
- 当前视图和设置

它不包含地图原图。这样文件更小，也避免把地图素材直接打包。

### 22. 导入全部场景

打开「更多 -> 导入全部场景」，选择 `.fogbundle.json`。

导入时，工具会要求你为场景重新选择本机地图。

要求：

- 地图尺寸需要匹配。
- 如果地图缺失或过大，可以走占位地图恢复流程。

如果导入包里记录了原当前场景，导入后会优先打开它。

### 23. 使用云存档

打开「更多 -> 云存档」。

云存档用于把进度存到你自己的私有 GitHub 仓库。

基本流程：

1. 准备 GitHub fine-grained PAT。
2. 输入云存档钥匙。
3. 连接私有仓库。
4. 点击“上传当前进度”。
5. 到另一台电脑打开同一网页。
6. 输入读取钥匙。
7. 点击“读取云端进度”。

云存档会上传：

- 状态数据
- 普通地图图片
- 能优化到阈值内的大地图图片

对于真正太大的地图，云存档会保存：

- 地图尺寸
- 地图指纹
- 迷雾
- Token
- 事件
- 标注
- 词条
- 揭示区域

然后在另一台电脑读取后，让你重新绑定同尺寸本机地图或云端地图库图片。

### 24. 绑定占位地图

如果读取云存档后出现占位地图，说明当前电脑没有对应大图本体。

编辑态可打开占位地图恢复向导：

- 绑定本机图
- 绑定云端地图库图片

绑定时必须尺寸匹配。尺寸不匹配时，工具会拒绝绑定，避免遮罩和对象错位。

### 25. 使用云端地图库

如果你把大图手动放进私有仓的 `manual-maps/` 目录，工具可以在占位地图恢复时从云端地图库选择图片。

可选的 `manual-maps/index.json` 可以提前提供：

- 图片路径
- 宽高
- 文件大小
- 备注

这可以避免下载几十 MB 后才发现尺寸不匹配。最终绑定时，工具仍会下载图片并校验真实尺寸。

## 数据与隐私

- 本地进度保存在当前浏览器的 IndexedDB。
- GitHub Pages 只提供静态网页。
- 工具没有项目作者服务器。
- 不使用账号系统。
- 不会把地图自动上传到公开仓库。
- `.fogbundle.json` 不包含地图原图。
- 如果使用云存档，状态和普通地图会上传到你自己控制的私有 GitHub 仓库。
- 清理浏览器站点数据会删除本机保存的项目。

## 离线使用

如果不想联网，可以下载 `index.html`，用浏览器直接打开。

离线使用时：

- 主功能仍可用。
- 本机保存仍可用。
- 导出 / 导入仍可用。
- 云存档不可用，因为它需要访问 GitHub API。

## 当前限制

- 不导出 PNG 成图，需要图片时请使用系统截图。
- 不做多人实时同步。
- 不做玩家权限模式。
- 不做自动视野、触发器、传送门等电子游戏逻辑。
- 单张地图最大边长约 4096 像素，超过会自动降采样。
- 标注按对象中心点判断是否被遮罩，不做逐像素裁切。
- 一个词条如果绑定多个揭示区域，一次揭示会产生多个撤销项，撤销时可能要按多次。
- 跑团态的揭示区域列表会显示区域名和格数，区域命名应避免剧透。
