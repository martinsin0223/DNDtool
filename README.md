# 战争迷雾地图工具 v3.5.0

给 DND / TRPG 跑团使用的共享桌面地图工具。核心用途是在同一块屏幕上展示地图、逐步揭示战争迷雾、管理事件提示、Token、网格遮罩和临时地图标注。

这是一个**单文件 HTML 工具**：无后端、无构建、无 npm/CDN、无外部依赖。双击 `index.html` 就能运行，也可以直接上传到 PushMe、GitHub Pages、Netlify、Vercel、Cloudflare Pages 等静态托管。

## 快速开始

1. 双击 `index.html`。
2. 点击「地图」加载 PNG / JPG / WebP 地图。
3. 初始迷雾选择「全部覆盖」。
4. 跑团时用笔刷逐步揭示地图。
5. 关闭浏览器即可，进度会自动保存到本机 IndexedDB。

## 核心功能

### 双层战争迷雾

工具有两层独立遮罩：

| 层 | 用途 |
|---|---|
| 地形层 | 遮住地图本身，控制玩家是否看见地形 |
| 事件层 | 遮住事件和部分标注，控制玩家是否看见提示点 |

笔刷支持三种模式：

| 模式 | 作用 |
|---|---|
| 揭 | 擦除当前层遮罩 |
| 盖 | 补回当前层遮罩 |
| 全揭 | 同时擦除地形层 + 事件层遮罩 |

地形层和事件层可以分别调整不透明度、颜色。事件层默认较低透明度，方便 DM 在准备期看清自己铺的事件区域。

### 事件系统

事件是地图上的剧情提示点，适合放机关、遭遇、线索、宝箱、地点说明。

- 支持默认 emoji、自定义字符、上传图片作为图标
- 每个事件包含标题和内容
- 可单击查看、双击编辑、拖动移动、Delete 删除
- 可设置为「受事件遮罩」或「永远可见」
- 事件内容是给 DM 的现场提示，不是电子游戏触发器

### 地图标注

v3.5.0 新增独立的地图标注层，和事件系统分开。

标注适合放：

- 地名
- 路线文字
- 阵营标识
- 手绘说明图
- 临时战术图标
- 贴在地图上的局部图片

支持两种标注：

| 类型 | 能力 |
|---|---|
| 文字标注 | 自定义文字、字号、颜色、移动、缩放 |
| 图片标注 | 上传 PNG / JPG / WebP，移动、缩放 |

每个标注有三种遮罩规则：

| 规则 | 效果 |
|---|---|
| 公开 | 不受迷雾影响，始终显示 |
| 受地形遮罩 | 地形层揭开后显示 |
| 受事件遮罩 | 事件层揭开后显示 |

标注不是事件，没有剧情卡片；它更像 PPT / 画布对象，负责直接贴在地图上。

### Token

Token 用于表示玩家队伍、敌人、NPC 或其他可移动棋子。

- 支持上传图片
- 圆形 / 方形 / 原图比例
- 可设置阵营描边
- 可拖动、滚轮缩放、双击编辑
- Token 永远在最上层，方便现场操作

### 六边形网格遮罩

可打开六边形网格，让笔刷按整格揭示/覆盖。

- 支持平移校准
- 支持大小调整
- 支持旋转校准
- 可把工具网格对准原图自带 hex 网格
- 网格设置随场景保存，导出当前场景时也会保留

当前只实现 pointy-top hex。方格、flat-top、两点/三点自动拟合是后续候选功能。

### 多场景管理

一个浏览器本地项目可以存多个场景：

- 每个场景有自己的地图、迷雾、事件、Token、标注、网格设置
- 启动时自动恢复最近场景
- 场景管理入口在「⋯ → 场景管理」

注意：当前导出/导入仍是「当前场景」级别，不是一次导出全部场景。

### 导出 / 导入

「⋯ → 导出当前场景」会导出 `.fogsave.json`：

- 包含两层迷雾
- 包含 Token、事件、标注
- 包含视口、网格、主题等设置
- **不包含地图原图**

导入前需要先加载对应地图。工具会检查地图尺寸；指纹不一致但尺寸一致时允许强制导入，适合跨设备迁移。

| 键 | 功能 |
|---|---|
| `E` | 揭示当前层 |
| `R` | 覆盖当前层 |
| `Q` | 全揭：同时揭示地形层和事件层 |
| `1` / `2` / `3` | 笔刷小 / 中 / 大 |
| `F` | 切换演示模式 |
| `Space` + 左键拖 | 平移视口 |
| `Shift` + 左键点击 | 添加事件 |
| `Ctrl/Cmd + Z` | 撤销最近一笔迷雾、事件或标注操作 |
| `Delete` / `Backspace` | 删除选中的 Token / 事件 / 标注 |
| `Esc` | 取消选中 / 关闭弹窗 / 退出放置状态 |
| `Ctrl+Shift+T` | 主题 JSON 输入框（开发者） |

## 推荐工作流

### 准备期

1. 加载地图。
2. 选择「全部覆盖」开局。
3. 设置 hex 网格，需要时校准到原图网格。
4. 预放事件、Token、文字标注、图片标注。
5. 用遮罩规则决定哪些标注随地形层或事件层出现。
6. 导出当前场景，带到现场电脑。

### 跑团现场

1. 加载地图并导入 `.fogsave.json`。
2. 玩家探索时，用地形层逐步揭示路线。
3. 需要同时露出地形和事件时，使用「全揭」笔刷。
4. 玩家发现线索时，揭开事件层或单击事件卡片。
5. 临时说明可以放文字/图片标注，但复杂编辑建议留在准备期。

## 数据与隐私

- 数据保存到浏览器本机 IndexedDB，不上传服务器。
- v3 起本机 IndexedDB 会保存每个场景的地图 Blob，用于下次自动恢复。
- 导出的 `.fogsave.json` 不包含地图原图，减少体积和版权风险。
- 隐私模式或禁用 IndexedDB 时仍可临时使用，但刷新后可能丢失数据。
- 不支持 HEIC；iPad 相册图建议先截图或转 PNG/JPG。

## 已知限制

- 不导出 PNG 成图；需要图片时请用浏览器或系统截图。
- 不做多人实时同步。
- 不做 Player Mode；DM 和玩家看到的是同一套共享工具。
- 不做触发器、视野自动揭示、传送门等电子游戏逻辑。
- 单张地图最大边 4096 像素，超过会自动降采样。
- 文字/图片标注按对象中心点判断遮罩可见性，不做逐像素裁切。
- 当前导出是单场景导出，多场景整包导出还未做。
